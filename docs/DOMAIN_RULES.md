# Domain Rules

## 서비스 목적

이 서비스는 미용실 고객에게 안내문이나 멤버십 약관을 전송하고, 고객이 모바일 링크에서 내용을 확인한 뒤 서명하도록 만드는 운영 도구다.

핵심 목표는 아래 두 가지다.

1. 지점/디자이너 단위로 문서를 표준화해서 발급한다.
2. 고객이 본인 확인 후 전자서명을 남기게 한다.

## 주요 엔터티

### 지점

- 실제 오프라인 매장 단위다.
- 디자이너, 템플릿, 지점 마스터 권한의 기준이 된다.

### 디자이너

- 반드시 하나의 지점에 속한다.
- 문서 발급 시 담당자를 기록하기 위해 사용한다.

### 문서 템플릿

- 지점별로 관리되는 문서 원본이다.
- 고객명, 날짜, 디자이너명 같은 값은 발급 시 치환된다.
- 문서 템플릿 삭제는 hard delete가 아니라 soft delete다. 삭제 상태가 되어도 기존 문서는 그대로 유지된다.

### 알림톡 템플릿

- Bizgo 알림톡 관리 API로 등록되는 발송 템플릿이다.
- 문서 발급 시 문서 템플릿과 별도로 선택한다.
- 원격 검수 상태를 가진다.
- 원격 삭제 후에도 로컬에는 이력용 카탈로그를 남긴다.

### 문서

- 특정 고객에게 실제 발급된 1회성 기록이다.
- 공개 링크 토큰을 가진다.
- 발급 이후에는 템플릿 원본과 분리된 스냅샷 데이터로 취급한다.

### 관리자

- 카카오 계정을 기준으로 권한을 가진다.
- 권한은 통합 마스터, 지점 마스터, 일반 어드민의 세 종류다.

## 권한 모델

### 통합 마스터

- 식별 방식: `MASTER_KAKAO_ID`
- 지점/디자이너/문서 템플릿/알림톡 템플릿 관리 가능
- 모든 문서 관리 가능
- 관리자 권한 부여 가능
- 지점 마스터 지정 가능

주의:

- 통합 마스터는 `admin_users` 테이블 대신 환경변수 기준으로 판별한다.

### 지점 마스터

- 특정 지점에 소속된다.
- 자신의 지점만 대상으로 지점 정보 수정 가능
- 자신의 지점 디자이너 관리 가능
- 자신의 지점 문서 템플릿 관리 가능
- 자신의 지점 알림톡 템플릿 관리 가능
- 자신의 지점 문서 발급/조회 가능

제약:

- 문서 생성 시 지점이 고정된다.
- 다른 지점 디자이너나 템플릿은 사용할 수 없다.

### 일반 어드민

- 문서 발급/조회 가능
- 지점/디자이너/문서 템플릿/알림톡 템플릿/권한 관리 불가

## 로그인 승인 도메인

### 기본 원칙

- 카카오 로그인을 시도했다고 바로 관리자 권한이 생기지 않는다.
- 허용되지 않은 로그인도 버리지 않고 저장한다.
- 통합 마스터가 나중에 보고 권한을 부여할 수 있어야 한다.

### 현재 흐름

1. 사용자가 `/admin/login`에서 카카오 로그인 버튼을 누른다.
2. 카카오 OAuth를 거쳐 사용자 ID와 닉네임을 가져온다.
3. 서버는 해당 계정이 통합 마스터인지, `admin_users`에 등록된 계정인지 확인한다.
4. 결과와 관계없이 `login_attempt_logs`와 `login_attempts`에 시도를 남긴다.
5. 권한이 없으면 로그인 화면으로 돌려보내고 `pending_approval` 메시지를 보여준다.
6. 통합 마스터는 권한 관리 화면에서 권한 없는 로그인 시도 계정만 본다.
7. 통합 마스터는 닉네임을 보고 권한을 부여한다.

## 문서 발급 규칙

### 필수 입력

- 지점
- 문서 템플릿
- 알림톡 템플릿
- 문서 제목
- 날짜
- 고객 이름
- 휴대폰 뒷자리 4자리
- 담당 디자이너

### 선택 입력

- 알림톡 수신 전체 번호
- 알림톡 즉시 발송 여부

### 서버 검증

문서 발급 시 아래를 서버에서 다시 확인한다.

1. 지점이 존재하는지
2. 템플릿이 존재하는지
3. 디자이너가 존재하는지
4. 문서 템플릿의 `branch_id`가 선택 지점과 같은지
5. 알림톡 템플릿의 `branch_id`가 선택 지점과 같은지
6. 디자이너의 `branch_id`가 선택 지점과 같은지
7. 지점 마스터라면 선택 지점이 본인 지점과 같은지

## 템플릿 치환 규칙

현재 지원하는 치환값은 아래와 같다.

- `{{branch_name}}`
- `{{document_title}}`
- `{{date}}`
- `{{customer_name}}`
- `{{phone_last4}}`
- `{{phone_full}}`
- `{{designer_name}}`
- `{{document_url}}`

치환은 문서 발급 시점에 수행한다. 발급된 뒤에는 템플릿 본문이 바뀌어도 과거 문서는 바뀌지 않는다.

## 공개 서명 규칙

### 본인 확인

- 고객은 휴대폰 뒷자리 4자리로 본인 확인을 한다.
- 일치 전에는 문서 내용을 볼 수 없다.

### 열람 기록

- 4자리 검증 성공 시 `viewed_at`이 저장된다.
- 같은 문서를 여러 번 열어도 최초 열람 시각은 유지한다.

### 서명 저장

- 서명은 PNG data URL로 저장한다.
- 이미 `signed` 상태라면 중복 저장을 막고 기존 완료 상태를 반환한다.

## Bizgo 알림톡 규칙

### 발송 가능 조건

아래 값이 모두 있어야 즉시 발송이 가능하다.

- `recipient_phone`
- `notification_template.template_code`
- `notification_template.sender_key`
- `notification_template.message`
- `BIZGO_API_KEY`

추가 메모:

- 현재 UI는 서명 링크 발송에 맞춘 기본 필드 중심이다.
- 공식 문서에 있는 다중 버튼, quick reply, 파일 첨부 검수 요청까지는 아직 노출하지 않는다.

### 발송 실패 처리

- 문서 생성과 알림톡 발송은 분리되어 있다.
- 문서는 먼저 생성된다.
- 발송 실패 시 문서를 롤백하지 않고 `bizgo_status = failed`로 기록한다.

## 도메인 불변 조건

다음 조건은 앞으로 기능을 추가할 때도 유지하는 편이 안전하다.

1. 문서는 스냅샷 기반이어야 한다.
2. 지점 마스터는 자기 지점 경계를 넘을 수 없어야 한다.
3. 승인되지 않은 로그인은 사라지면 안 된다.
4. 공개 링크는 최소한의 본인 확인 절차를 거쳐야 한다.
5. 관리자 권한 결정은 서버에서만 수행해야 한다.

## 아직 없는 것

- 감사 로그 조회 화면
- 서명 이미지 외부 스토리지 분리
- 템플릿 버전 관리
- 문서 취소/폐기 상태
- 재발송 기능
- 테스트 코드
- migration 버전 관리

이 영역들은 다음 작업자가 확장 대상으로 보기 쉽다.
